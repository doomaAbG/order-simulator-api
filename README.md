# Multithreaded Order Simulator API 🍔🚀

Высокопроизводительный бэкенд-симулятор обработки заказов (торговая площадка / кухня ресторана), написанный на современном **C++20** с использованием веб-фреймворка **Crow** и СУБД **PostgreSQL**.

## 🏗️ Архитектура и Особенности
* **Многопоточный HTTP-сервер:** Архитектура Crow обрабатывает входящие клиентские запросы (REST API) в пул-потоков.
* **Асинхронный Background Worker:** Реализован независимый фоновый поток (`std::thread`), выполняющий роль обработчика очереди заказов. Он автоматически сканирует БД, изолирует новые заказы и симулирует их жизненный цикл.
* **Data Safety (Блокировки БД):** Для предотвращения эффекта "Race Condition" и конфликтов между потоками при выборке заказов используется безопасный SQL-механизм: `FOR UPDATE SKIP LOCKED`.
* **ACID Транзакции:** Плотная интеграция с PostgreSQL через `libpqxx` гарантирует атомарность списания остатков товаров со склада при создании заказов.

## 🛠️ Технологический стек
* **Язык:** C++20 (MSVC / GCC)
* **Веб-фреймворк:** Crow (Header-only, Fast HTTP Server)
* **База данных:** PostgreSQL 16
* **Драйвер БД:** libpqxx (C++ клиент для PostgreSQL)
* **Пакетный менеджер:** vcpkg
* **Система сборки:** CMake

## 🚦 REST API Эндпоинты

| Метод | Эндпоинт | Описание |
| :--- | :--- | :--- |
| **GET** | `/products` | Получить актуальный список товаров, их цены и остатки на складе |
| **POST** | `/orders` | Создать новый заказ (автоматически получает статус `PENDING`) |
| **GET** | `/orders-history` | Посмотреть историю всех заказов и их текущие статусы |
| **PATCH** | `/orders/status` | Ручное изменение статуса заказа (для администратора) |

---

## 🔄 Жизненный цикл заказа (Симуляция)

1. Клиент делает `POST /orders` $\rightarrow$ Сервер списывает `stock` товара и сохраняет заказ в БД со статусом **`PENDING`**.
2. Фоновый поток (`Worker Thread`) перехватывает заказ $\rightarrow$ Меняет статус на **`PROCESSING`** (имитация сборки/готовки 5 секунд).
3. По истечении таймера воркер переводит заказ в статус **`COMPLETED`**.

## 🚀 Как запустить проект локально

### 1. Подготовка Базы Данных
Создайте базу данных `order_simulator` в PostgreSQL и выполните в ней следующие SQL-скрипты:

```sql
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price NUMERIC(10, 2) NOT NULL,
    stock INT NOT NULL
);

CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    product_id INT REFERENCES products(id),
    quantity INT NOT NULL,
    total_price NUMERIC(10, 2) NOT NULL,
    status VARCHAR(20) NOT NULL
);

INSERT INTO products (name, price, stock) VALUES 
('Burger', 5.99, 100),
('Fries', 2.99, 150),
('Cola', 1.99, 200);
