# Order Simulator API

Высокопроизводительный микросервис для симуляции и обработки заказов, написанный на **C++** с использованием архитектуры событийно-ориентированного программирования (EDA).

## 🛠 Технологический стек
* **Бэкенд:** C++ (Crow framework для HTTP REST API)
* **База данных:** PostgreSQL (хранение состояния заказов и продуктов)
* **Брокер сообщений:** Apache Kafka (публикация событий о созданных заказах)
* **Тестирование:** Python (пакеты `pytest` и `requests` для интеграционного тест-сьюта)

## 🏗 Архитектура системы
1. Клиент отправляет HTTP POST запрос на добавление заказа.
2. Crow-сервер валидирует JSON и передаёт данные в слой данных.
3. `OrderRepository` выполняет транзакцию в PostgreSQL, проверяя внешние ключи (Foreign Keys).
4. После успешной записи `OrderPublisher` отправляет событие с ID заказа в топик Kafka для последующей обработки консьюмерами.
